# 代码风格规范

## Python 代码规范

1. 遵循 PEP 8 规范
2. 缩进使用 4 个空格
3. 每行代码不超过 88 个字符
4. 函数和类之间空两行
5. 导入顺序：标准库 > 第三方库 > 本地应用/库
6. 使用有意义的变量和函数命名，采用 snake_case 风格
7. 类名采用 PascalCase 风格
8. 常量命名使用全大写 + 下划线

## 数据库规范

1. 表名使用小写 + 下划线
2. 字段名使用小写 + 下划线
3. 主键统一命名为 id
4. 时间字段统一使用 create_time, update_time 等命名
5. 外键命名格式：{关联表名}_id

## API 规范

1. URL 路径使用小写 + 下划线
2. HTTP 方法使用规范：
   - GET: 获取资源
   - POST: 创建资源
   - PUT: 更新资源（全量）
   - PATCH: 更新资源（部分）
   - DELETE: 删除资源
3. 响应格式统一为 JSON:
   ```json
   {
     "code": 200,
     "message": "操作成功",
     "data": {}
   }
